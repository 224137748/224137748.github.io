# 原型链

  • 概念： JS里面的对象可能会有父对象，父对象还有父对象...当获取对象某个属性时，如果该对象没有属性值，则会通过__proto__指针向父对象的prototype对象中查找....
  • 根本： 继承
    • 属性： 对象中几户都会有一个__proto__属性，这个属性指向他的父对象
      • 意义： 可以实现让该对象访问到父对象的中的属性和方法；
          function Person() {};
    var p1 = new Person();
    p1.__proto__ ==== Person.prototype // true
  • 跟对象： Object.prototype
    • var arr = [1, 2, 3];
    • arr.proto: Arr.rptotype
    • arr.proto.__proto__找到跟对象
  • 错误的理解： 万物皆继承自对象？应该是继承自Object.prototype

# 变量作用域

  • 变量作用域的概念：就是一个变量可以使用的范围
  • JS中首先有一个最外层的作用域：称之为全局作用域
  • JS中还可以通过函数创建出一个独立的作用域，其中函数可以嵌套，所以作用域也可以嵌套
var age=18;     //age是在全局作用域中声明的变量：全局变量

function f1(){
    console.log(name);      //可以访问到name变量
    var name="周董" //name是f1函数内部声明的变量，所以name变量的作用域就是在f1函数内部

    console.log(name);      //可以访问到name变量

    console.log(age);       //age是全局作用域中声明的，所以age也可以访问
}

console.log(age);       //也可以访问

    //多级作用域
    //-->1级作用域
    var gender="男";
    function fn(){
        //问题：
        //gender:可以访问
        //age:  可以访问
        //height:不能访问

        //-->2级作用域
        return function(){
            //问题：
            //gender:   通过一级一级作用域的查找，发现gender是全局作用域中声明的变量
            //age:
            //height：
            console.log(gender);

            //-->3级作用域
            var height=180;
        }
        var age=5;
    }

# 作用域链

  • 由于作用域是相对于变量而言的，而如果存在多级作用域，这个变量又来自于哪里？这个问题就需要好好地探究一下了，我们把这个变量的查找过程称之为变量的作用域链

  • 作用域链的意义：查找变量（确定变量来自于哪里，变量是否可以访问）

  • 简单来说，作用域链可以用以下几句话来概括：(或者说：确定一个变量来自于哪个作用域)

    • 查看当前作用域，如果当前作用域声明了这个变量，就确定结果
      • 查找当前作用域的上级作用域，也就是当前函数的上级函数，看看上级函数中有没有声明
        • 再查找上级函数的上级函数，直到全局作用域为止
          • 如果全局作用域中也没有，我们就认为这个变量未声明(xxx is not defined)

  • 难点：何如查看一个函数的上级作用域？*****

    • 看上级作用域不是看函数在哪里调用，而是看函数在哪里编写。
    • 分析：js解析引擎会对function关键词提前预解析，将function函数保存在内存中，如果函数内存在变量的使用，那么该变量则为函数定义时上下文内的变量；

  • 举例1：

    var name="张三";
    function f1(){
        var name="abc";
        console.log(name);
    }
    f1();
  • 举例2：
    var name="张三";
    function f1(){
        console.log(name);
        var name="abc";
    }
    f1();
  • 举例3：
    var name="张三";
    function f1(){
        console.log(name);
        var name="abc";
    }
    f1();
  • 举例4：
    var name="张三";
    function f1(){
        return function(){
            console.log(name);
        }
        var name="abc";
    }
    var fn=f1();
    fn();
  • 举例5：
    var name="张三";
    function f1(){
        return {
            say:function(){
                console.log(name);
                var name="abc";
            }
        }
    }
    var fn=f1();

# 函数的4种调用方式

  • 1、函数调用
  • 函数的this指向window
    var age=18;
    var p={
        age:15
        say:function(){
            console.log(this.age);
        }
    }
    var s1=p.say
    s1();       //函数调用
  • 2、方法调用
  • 函数的this指向对象本身
    var age=18;
    var p={
        age:15
        say:function(){
            console.log(this.age);
        }
    }
    p.say()//方法调用
  • 3、new调用(构造函数)
  • 函数的this指向实例本身
    var age=18;
    var p={
        age:15
        say:function(){
            console.log(this.age);
        }
    }
    new p.say()//构造函数调用
  • 4、上下文方式(call、apply、bind)
  • 函数的this为方法传入的执行上下文
    var length=21;
    function f1(){
        console.log(this.length);
    }
    f1.call([1,3,5])
    f1.apply(this)
    f1.call(5)
  • 在ES6的箭头函数之前的时代，想要判断一个函数内部的this指向谁，就是根据上面的四种方式来决定的
